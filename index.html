휴가대장 만들기

파일 2개를 올리고 연도를 선택하면 버튼이 활성화됩니다.
계산 방식과 시트 서식(색상·테두리·병합·표시형식)은 기존 휴가대장(2026) 시트와 동일합니다.
생성된 시트의 숫자는 계산 결과가 고정된 값입니다. 데이터가 바뀌면 다시 생성해 주세요.

사용 설명서

처음 시작하기

① 인원DB 양식 받기 / 휴가DB 양식 받기로 엑셀 양식을 내려받아 데이터를 채웁니다.
② 1단계에 두 파일을 각각 올립니다.
③ 연도를 고르고 엑셀 파일 생성을 누르면 끝!

기존에 쓰던 통합 엑셀(인원DB·휴가DB 시트 포함)이 있다면 그 파일 하나를 양쪽에 똑같이 올려도 됩니다. 필요한 시트를 자동으로 찾아냅니다.

자동 저장

파일을 올리거나 데이터를 수정하면 이 브라우저에 자동 저장됩니다. 다음에 이 파일을 다시 열면 마지막 데이터가 그대로 복원되어, 파일을 매번 다시 올릴 필요가 없습니다.

  • 같은 PC · 같은 브라우저에서만 유지됩니다.
  • 화면 위 상태바의 초록 점이 켜져 있으면 저장이 동작 중입니다.
시크릿 창에서 열거나, 브라우저의 "인터넷 사용 기록 삭제"로 사이트 데이터를 지우면 저장된 내용이 사라집니다. 중요한 시점마다 백업 내려받기를 해 두세요.

백업과 복원

백업 내려받기 — 현재 인원DB·휴가DB 전체를 JSON 파일 하나로 저장합니다. 다른 PC로 옮기거나 보관용으로 쓰세요. 저장할 때 비밀번호를 입력하면 파일이 AES-256으로 암호화되어, 비밀번호를 모르면 열 수 없습니다(인사 정보 보호). 비밀번호를 비우면 암호화 없이 저장됩니다.
백업 복원 — 백업 JSON 파일을 선택하면 그 시점의 데이터로 되돌립니다. 암호화된 파일이면 비밀번호를 묻고, 맞으면 복원됩니다. (현재 데이터는 덮어써집니다)
저장 데이터 초기화 — 브라우저에 저장된 데이터를 지우고 처음 상태로 돌아갑니다.

암호화 백업은 비밀번호를 잊으면 복구할 수 없습니다. 비밀번호는 별도로 안전하게 보관하세요. 메일·공유폴더로 백업 파일을 주고받을 때는 암호화 저장을 권장합니다.

데이터 수정 (2단계)

  • 셀을 클릭해 바로 수정합니다. 수정 즉시 자동 저장됩니다.
  • ＋ 추가 버튼으로 인원/휴가기록을 추가, 행 끝 ✕로 삭제합니다.
  • 휴가DB 탭의 검색창에서 성명·연도·휴가구분으로 찾을 수 있습니다.
  • 휴가내역 합계 탭 — 사람별 발생연차·사용연차·잔여연차 총계를 보여줍니다. 기준(입사일/회계년도)과 기준일을 바꿔 계산할 수 있고, 이름으로 검색할 수 있습니다. 행을 클릭하면 그 사람의 계산 과정이 표로 펼쳐집니다. 엑셀 생성 시 '휴가내역' 시트로도 같은 기준·기준일로 함께 들어갑니다.
  • 수정한 인원DB/휴가DB 내려받기로 엑셀 파일로도 보관할 수 있습니다.

날짜 · 입력 형식

  • 날짜는 2026-06-12 형식 (2026.06.12, 2026/06/12도 인식)
  • 퇴사일이 없으면 - 또는 빈칸
  • 사용일은 숫자 — 하루 1, 반차 0.5
  • 휴가DB의 YEAR를 비워두면 신청일의 연도가 자동 적용됩니다.

발생연차 기준 — 입사일 vs 회계년도

휴가내역 합계 탭에서 두 가지 방식으로 발생연차를 볼 수 있습니다. 사용·잔여의 집계 범위(전체 누적)는 같고, 발생연차 계산만 달라집니다.

방식발생연차 계산
입사일 기준입사일부터 기준일까지 만 1년마다 발생한 연차의 누적
회계년도 기준입사 첫해는 다음 해 1/1에 비례연차(입사일~12/31 일수 ÷ 365 × 15일, 0.5일 단위 올림) 지급, 이후 매년 1/1 발생. 여기에 입사 1년차 월차(최대 11일)를 더함

두 방식 모두 발생연차에 입사 1년차 월차를 포함합니다. 회계년도 기준은 근로기준법 행정해석을 따른 것으로, 입사 첫해 비례분 때문에 신입은 더 많게, 장기근속자는 입사일 기준과 비슷하거나 적게 나올 수 있습니다.

퇴직 정산 주의 — 법상 퇴직 시점에는 입사일 기준으로 재계산하여 더 유리한 쪽으로 정산해야 합니다. 이 도구의 회계년도 값은 평상시 관리용 참고치이며, 퇴직 정산은 입사일 기준 값과 함께 확인하세요.

합계 탭에서 행을 클릭하면 그 사람의 발생·사용·잔여가 어떤 항목으로 계산됐는지 한 줄씩 펼쳐 보여줍니다.

노란 경고 상자의 의미

  • 기타휴가로 집계되는 휴가구분 — 표준 구분명과 다르게 입력된 항목입니다. 의도한 것이면 그대로 두고, 아니면(예: 경조사 → 경조휴가) 2단계에서 수정하세요.
  • 인원DB에 없는 성명 — 휴가기록은 있는데 인원DB에 없는 사람입니다. 대장에 나타나지 않으니 인원DB에 추가하거나 이름 오타를 확인하세요.

자주 묻는 질문

Q. 저장이 안 된다고 떠요.
이 파일을 PC에 내려받아 더블클릭으로 열어 주세요. 일부 미리보기 환경이나 보안 설정에서는 브라우저 저장이 막혀 있습니다. 그 경우에도 백업 내려받기/복원으로 데이터를 이어갈 수 있습니다.

Q. 다른 PC에서 쓰려면?
이 HTML 파일과 백업 JSON을 함께 옮긴 뒤, 새 PC에서 백업 복원을 누르세요.

Q. 생성된 엑셀 숫자가 안 바뀌어요.
생성 파일은 계산 결과가 고정된 값입니다. 데이터 수정 후 다시 생성해 주세요.
